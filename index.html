Connect in three steps

AntSeed buyer auto-selection is disabled in the default flow. Install the buyer, fund it, then pin this peer explicitly.

Step 01

Install and start

Run the local OpenAI-compatible buyer proxy on port 8377.

npm install -g @antseed/cli
antseed buyer start
Step 02

Fund the buyer

Open the payment portal and deposit USDC on Base.

antseed payments
antseed buyer balance
Step 03

Pin this provider

The pin persists across buyer-proxy restarts.

antseed buyer connection set --peer c50de6922b00677c93007c01924586de887ced7b
curl -s http://127.0.0.1:8377/v1/models | jq '.data[].id'

Use your existing tools

All examples assume the buyer proxy is running and this peer is pinned.

OpenAI-compatible clients

Point any OpenAI Chat Completions client at the local proxy.

curl http://127.0.0.1:8377/v1/chat/completions \
  -H "content-type: application/json" \
  -d '{"model":"opus-4.7","messages":[{"role":"user","content":"Hello"}]}'

Claude Code

The AntSeed wrapper configures the local proxy and placeholder API key.

npm install -g @anthropic-ai/claude-code
antseed claude --model opus-4.7

Pi coding agent

Install the official extension, reload Pi, then choose an AntSeed model.

npm install -g @mariozechner/pi-coding-agent
pi install git:github.com/AntSeed/pi-antseed
pi
# In Pi: /reload
# Then:  /model antseed/opus-4.7

LangChain Python

Use ChatOpenAI with the local AntSeed base URL.

from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    model="opus-4.7",
    base_url="http://127.0.0.1:8377/v1",
    api_key="antseed",
)
print(llm.invoke("Hello").content)

Hermes agent

Register AntSeed as a custom Chat Completions provider.

model:
  default: opus-4.7
  provider: antseed
custom_providers:
  - name: antseed
    base_url: http://127.0.0.1:8377/v1
    api_key: antseed-p2p
    api_mode: chat_completions
    models:
      - opus-4.7
      - opus-4.8
      - claude-sonnet-5
